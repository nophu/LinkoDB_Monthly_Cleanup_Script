Linko DBMonthly Cleanup

An automated quality-check pipeline that reads the monthly Linko report exports, validates every field against the program rubric, and produces a clean, color-coded report of exactly what needs fixing.

read rubric + reports validate each field auto-fix what's certain flag what needs review export Excel report
What it checks

Five reports, one pass

01

Permit List

Extractor ID, Extractor Type, Trap Size & Units, Cleaning Frequency.

02

FSE Inspections

Receiving Plant, Class Code, Second Class, Trunk Line.

03

Master List

Map Category against the approved values.

04

AG Extract Summary

Extractor ID and Extractor Type.

05

Inspection Events

Event Type against the current pick list.

How to read the output

Two colors, zero guesswork

Green — Auto-fixed

A clear correction the tool applied for you (casing, EX prefix, trap unit). Apply it as-is in Linko.

Yellow — Needs review

No certain fix; a person decides (old extractor IDs, unknown values, retired plants).

Running it

From export to report

  1. Pull the five monthly report exports out of Linko (.xlsx or .xls).
  2. Launch the tool and pick the rubric file when prompted.
  3. Select the report file(s) to check — one or several at once.
  4. The tool validates every field and writes Monthly_Quality_Check_Report.xlsx.
  5. Open the report, work the green rows first, then review the yellow ones.
